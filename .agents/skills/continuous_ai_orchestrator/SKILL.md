# Continuous AI Orchestrator (v2.1)

## Purpose

The Continuous AI Orchestrator is the "Ecosystem Brain" responsible for autonomous lifecycle management. It dynamically dispatches specialized skills Based on an **Impact Matrix**, ensuring every modification is validated for security, documentation parity, and architectural integrity.

## Inputs / Outputs

### Inputs
- `change_set` (Git Diff): The raw source of change.
- `trigger_event` (Enum): `COMMIT`, `PR_OPEN`, `MANUAL_RELEASE`, `PILOT_UPDATE`.
- `impact_matrix` (JSON): Component-level mapping matching the schema: `{ "components": [{ "path": string, "domain": string, "risk_level": int }] }`.

### Outputs
- `orchestration_summary` (Artifact): Detailed log of triggered skills and their results.
- `gate_status` (Enum): `SUCCESS` | `FAIL` | `DEGRADED`.
- `pipeline_id` (String): Unique UUID for the execution trace.

## Preconditions
- `impact_matrix` MUST be generated by `change-impact-visualizer`.
- Repository MUST be in a clean git state (verified via `ai-development-protocol`).

---

## Instructions

### 1. Change Sensitivity Analysis
**Dependency**: `change-impact-visualizer`
- Execute `change-impact-visualizer` on the `change_set`.
- **Validation**: If the visualizer fails, the orchestrator MUST default to the `CORE_SECURITY` profile as a fail-safe.

### 2. Profile Selection
Map the `impact_matrix` to an **Execution Profile**:

| Impact Domain | Profile | Mandatory Skills |
| :--- | :--- | :--- |
| **Engine / Sombra** | `CORE_SECURITY` | `ai-development-protocol`, `secret-scanner`, `security-advisory-scanner`, `red-team-evasion-scanner` |
| **PII / Policy** | `PRIVACY_COMPLIANCE`| `policy-schema-generator`, `policy-impact-simulator`, `compliance-integrity-suite` |
| **Docs Only** | `DOCS_SYNC` | `compliance-docs-orchestrator` |
| **Dashboard** | `UI_INTEGRITY` | `enterprise-dashboard-integrity-checker`, `ui-regression-orchestrator` |
| **Unknown** | `DEFAULT` | `secret-scanner`, `architectural-linter`, `compliance-docs-orchestrator` |

### 3. Execution DAG

#### Phase A: Global Sanity (Sequential)
1. **Ecosystem State Tracker**: Perform redundancy check. Skip sub-tasks if valid cache exists.
2. **Secret Scanner**: Universal scan for credentials and internal paths.

#### Phase B: Domain Execution (Parallel)
Execute all skills defined in the selected Profile. Each skill MUST return a signed `validation_receipt`.

#### Phase C: Finishing & Redaction (Sequential)
1. **Content Redactor**: Sanitize the artifact staging area.
2. **Integrity Check**: verify all `validation_receipt` signatures against the Ecosystem Key.

### 4. Post-Flight Assertions
- `gate_status` == `SUCCESS` only if ALL mandatory skills returned `SUCCESS`.
- `orchestration_summary` MUST be written to `.agents/logs/orchestration/{{pipeline_id}}.md`.

## Failure Handling
- **Security Breach**: If `red-team-evasion-scanner` returns `FAIL`, trigger `LOCKDOWN` (abort release, notify CISO).
- **Dependency Missing**: If a required skill is missing, halt and request `repository-knowledge-map` to locate alternatives.

## Postconditions
- All generated artifacts MUST be registered in the `ecosystem-state-tracker`.
- `gate_status` MUST be available for CI/CD consumption via `status.json`.
