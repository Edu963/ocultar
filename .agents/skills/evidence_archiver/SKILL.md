# Evidence Archiver (v1.0)

## Purpose

The Evidence Archiver transforms transient compliance reports into permanent, verifiable audit evidence. It provides the "Accountability" required by GDPR Article 5(2) and ISO 27001 by cryptographically binding the report to the Ocultar system identity.

## Prerequisites

- **Signing Utility**: Access to the `crypto/ed25519` logic found in `verify_key.go`.
- **Ocultar Master Key**: Access to the environment variables required for key derivation (HKDF).
- **Validator Output**: A `compliance_audit_report` generated by the `audit-log-validator`.

## Inputs / Outputs

### Inputs:
- `raw_evidence` (Artifact): The report or log data to be archived.
- `metadata` (JSON): Contextual data (Timestamp, User, System Version).
- `signing_key_id` (String): Reference to the Ed25519 key pair to be used.

### Outputs:
- `signed_evidence_package` (File): A `.zip` or `.tar.gz` archive containing the report, its hash, and the detached signature.
- `verification_snippet` (Code): A command or script to verify the package integrity.

---

## Instructions

### Step 1 – Canonicalization
- Normalize the `raw_evidence` into a deterministic string (e.g., sorted JSON keys) to ensure stable hashing.
- Append the `metadata` to the evidence to create the "Archive Manifest".

### Step 2 – Hashing & Fingerprinting
- Generate a SHA-256 hash of the Archive Manifest.
- This fingerprint serves as the unique ID for the evidence package.

### Step 3 – Cryptographic Signing (Accountability binding)
- Retrieve the system's Ed25519 private key (derived via HKDF from `OCU_MASTER_KEY`).
- Sign the SHA-256 fingerprint using Ed25519.
- **Validation**: Confirm the signature length and format (Base64).

### Step 4 – Package Assembly
- Bundle the following into a timestamped directory:
    - `manifest.json`: The original evidence and metadata.
    - `signature.sig`: The detached Ed25519 signature.
    - `verifier.go`: A copy of `verify_key.go` or a similar utility for third-party auditors.

### Step 5 – Integrity Proof Generation
- Generate a final receipt containing the HASH and the SIGNATURE for the `CISO Dashboard`.

---

## Failure Handling

- **Key Derivation Failure**: If the Ed25519 key cannot be derived from the master key, halt and alert the **Security Administrator**.
- **Malformed Evidence**: If the input is not a valid artifact, request a re-run of the validator.

## Examples

### Example 1: SOC2 Audit Package
- **Input**: Quarterly compliance report `compliance_q1.json`.
- **Action**: Sign with Ed25519; package with metadata.
- **Output**: `ocultar_evidence_2026_Q1.tar.gz`.

### Example 2: Real-time Incident Archival
- **Input**: A detected PII leak event from the audit log.
- **Action**: Immediately archiver the surrounding log lines as forensic evidence.
- **Result**: Signed evidence proving the system successfully blocked the leak.
